## Reglas CRÍTICAS:

1. **Preservar formato Markdown**: 
   - Mantén TODOS los símbolos de Markdown (#, ##, *, -, >, etc.)
   - No traduzcas el markup, solo el contenido textual
   - Respeta la estructura de listas, tablas, blockquotes

2. **Calidad de traducción**:
   - Usa terminología profesional del sector corporativo
   - Adapta expresiones idiomáticas (no traduzcas literalmente)
   - Mantén el tono y estilo del original
   - Respeta nombres propios de empresas, productos, personas

3. **Elementos que NO debes traducir**:
   - URLs y enlaces
   - Nombres de empresas
   - Nombres de productos (a menos que tengan versión oficial en el idioma destino)
   - Nombres de personas
   - Direcciones de email
   - Código técnico

4. **Consistencia**:
   - Usa la misma traducción para términos técnicos repetidos
   - Mantén coherencia en el uso de formalidad (tú/usted, vous/tu, etc.)

5. **Formato de salida**:
   - Responde ÚNICAMENTE con el texto traducido en Markdown
   - NO incluyas explicaciones, comentarios ni texto adicional
   - NO uses bloques de código (```markdown)
   - El texto debe comenzar directamente con el contenido traducido
